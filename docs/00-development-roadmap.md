# Development Roadmap

This roadmap keeps the Thrive AI project organised as it develops. Each stage should be updated as the idea becomes more complete.

## 1. Project Concept

Define what Thrive AI is, who it helps, why it matters, and what makes it personal.

Status: In progress

Current documents:

- `docs/01-project-concept.md`
- `docs/02-user-personas-and-use-cases.md`

## 2. Features

Define the main user-facing features, including job support, confidence coaching, wellness support, assignment planning, reminders, and progress tracking.

Status: Not started

## 3. System Design

Design the technical structure of the product, including frontend, backend, AI agent layer, tools, memory, safety checks, and database.

Status: Not started

## 4. Data Storage and Privacy

Define what data is collected, how it is stored, how user consent works, how privacy is protected, and how users can delete their data.

Status: Not started

## 5. Real-Life Impact

Explain how Thrive AI can support real users, what problems it can reduce, and what ethical boundaries it must respect.

Status: Not started

## 6. Future Development

Plan how the project can grow over time with voice support, document uploads, reminders, deeper personalization, integrations, and more user modes.

Status: Not started

## Session Rule

At the end of every working session, update:

- `docs/progress-log.md`
- A session note inside `docs/session-notes/`
- `docs/decision-log.md` if any important decision was made

This is important because Thrive AI should have a visible development history that shows planning, effort, reasoning, iteration, and improvement.
